## 1. Guardrail Implementation (Harden Runtime)

The following guardrails were implemented to enforce the cloud-native operating model and prevent silent failures during dispatch or unauthorized stage execution.

### 1A. Cloud-Serve Fail-Closed Startup
- **File**: `backend/app_factory.py`
- **Logic**: Added `_validate_cloud_dispatch_config()`.
- **Behavior**: If `APP_RUNTIME_ROLE=cloud-serve` and `DATA_BACKEND=neon`, the application now raises a `RuntimeError` during startup if `CLOUD_RUN_PROJECT_ID`, `CLOUD_RUN_REGION`, or `SERVE_REFRESH_CLOUD_RUN_JOB_NAME` are missing.
- **Pre-Deployment Fix**: Verified that `ceiora-prod-serve` was missing these variables. Patched the Cloud Run service configuration manually to prevent a crash loop upon deployment.

### 1B. Cloud-Job Stage Restriction (with Emergency Override)
- **File**: `backend/orchestration/run_model_pipeline.py`
- **Logic**: Added `_validate_cloud_job_role()`.
- **Behavior**: 
    - `ingest` is **strictly forbidden** in `cloud-job` mode.
    - `source_sync` is forbidden by default, but can be enabled via `ORCHESTRATOR_ALLOW_EMERGENCY_SYNC=true` for self-healing in the cloud (e.g., re-syncing Neon if a local sync was missed).
- **Verification**: Verified via `test_cloud_job_blocks_ingest_stage`, `test_cloud_job_blocks_source_sync_stage_by_default`, and `test_cloud_job_allows_source_sync_if_emergency_toggle_on`.

## 2. Diagnostics Read Path Remediation (Operator Workflow)

To address the `503 cache_not_ready` blocker identified in the previous evidence slice (2026-04-13), the Neon read path was hardened to expose underlying errors while protecting secrets.

### 2A. Neon Read Transparency & Sanitization

- **Change**: Updated `load_current_payloads_neon` to log exceptions (Neon connection failures or query errors) instead of silently returning `None`.
- **Security**: Added explicit sanitization to the logger to ensure that if a DSN or credential is present in the exception message (common in `psycopg` connection errors), it is masked before hitting Cloud Logging.
